# Prometheus, Loki, Grafana Stack with Log Generator
- This application creates logs for Loki to analyse. It uses the Allam-2-7b model to generate logs in a specific format and saves them to a file.
- The application runs indefinitely, generating logs at random intervals between 1 and 5 seconds. The logs are concise and relevant to software development, and they are categorized into different log types such as INFO, WARNING, ERROR, DEBUG, CRITICAL, EXCEPTION, ALERT, and EMERGENCY. 
- The application is designed to be used in a Kubernetes environment, where it can be deployed as a pod and can be configured to use a secret for the OpenAI API key. The generated logs can then be collected and analyzed by Loki, which is a log aggregation system that is part of the Grafana ecosystem. This setup allows for efficient log management and analysis, making it easier to monitor and troubleshoot applications.

## Instructions of use
1. Clone the repository to your local machine.
2. Navigate to the `k8s/log-generator` directory and open the `secrets.yaml` file. Replace `YOUR_OPENAI_API_KEY_HERE` with your actual OpenAI API key.
3. Apply the Kubernetes manifests in the following order:
   - First, add your log directory path to the `hostPath` field in `k8s/shared/pv.yaml`
   - Apply the PersistentVolume and PersistentVolumeClaim defined in `k8s/shared/pv.yaml` to mount the log directory.
   - Next, apply the Secret defined in `k8s/log-generator/secrets.yaml` to create the secret that will be used by the log generator application.
   - Finally, apply the Deployment defined in `k8s/log-generator/deployment.yaml` to deploy the log generator application in your Kubernetes cluster.
4. Once the application is running, it will start generating logs in the specified log file.
5. Configure Promtail to read the logs from the log file generated by the application and send them to Loki for aggregation and analysis using the `promtail-config.yaml` file and deploy it using the `promtail-daemonset.yaml` file.
6. Configure Loki using the `loki-config.yaml` file to collect logs from the log file generated by the application and deploy it using the `loki-dep.yaml` file.
7. Finally, configure Grafana to visualize the logs collected by Loki using the `grafana-dep.yaml` file.
